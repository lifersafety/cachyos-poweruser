04 How the offline patch works
--- a/etc/calamares/settings.conf (official CachyOS) +++ b/etc/calamares/settings.conf (cachyos-poweruser) @@ -exec sequence @@ - packages - displaymanager +- shellprocess@offline_detect # detect bundled localrepo, inject into pacman.conf - packages # unchanged — works offline or online transparently +- shellprocess@restore_online # remove localrepo from installed system - bootloader - umount
1

Boot — localrepo auto-activates

A systemd service runs at live-ISO boot. If /var/cache/localrepo exists (offline ISO), it adds [cachyos-local] to pacman.conf with highest priority and writes a flag file.

2

Install — Calamares runs unchanged

The shellprocess@offline_detect step reads the flag and ensures the target's pacman.conf also has the local repo. All other Calamares modules are 100% official CachyOS.

3

Package install — transparent fallback

Pacman serves packages from the bundled repo first. Any package not in the local repo falls back to online mirrors automatically. No changes to the packages list.

4

Post-install — online repos restored

The shellprocess@restore_online step removes [cachyos-local] from the installed system. First boot gets standard CachyOS mirrors. pacman -Syu works normally.

05 Get started

ONE-LINE INSTALL (Linux with dialog installed):

bash
# Opens TUI menu — build ISO, install, or download
$ curl -fsSL https://raw.githubusercontent.com/lifersafety/cachyos-poweruser/main/setup.sh | bash

BUILD ISO MANUALLY (Arch/CachyOS host required):

bash
# Install build deps
$ sudo pacman -S --needed archiso git dialog

# Clone cachyos-poweruser
$ git clone https://github.com/lifersafety/cachyos-poweruser
$ cd cachyos-poweruser

# Build KDE offline ISO (clones official CachyOS-Live-ISO + patches)
$ sudo bash build/build-iso.sh --profile kde-offline --method native

# Write to USB
$ sudo dd if=~/.cachyos-poweruser/out/*.iso of=/dev/sdX bs=4M status=progress oflag=sync

INSTALL FROM LIVE ISO (booted into target machine):

bash — live ISO
